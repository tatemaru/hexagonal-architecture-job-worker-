## Context

既存のバックエンド技術書（`docs/backend/02_ddd.md`, `03_hexagonal.md`）は概念の導入はあるが、学習者が「なぜそう設計するのか」「どう読み解けばよいか」を深く理解するには不足している。Alistair Cockburn の Hexagonal Architecture をベースに、用語の定義・意図・変形パターン・図解を補強する必要がある。DDD も概念の相互関係や境界の考え方が薄いため、厚めに再構成する。

## Goals / Non-Goals

**Goals:**
- 既存の DDD/ヘキサゴナル章を増補し、学習導線が明確な技術書レベルにする
- Cockburn の Hexagonal Architecture の主要ポイントを噛み砕いて説明する
- Mermaid 図を追加し、概念と実装の対応を視覚化する

**Non-Goals:**
- アプリの実装や API を変更する
- OpenSpec の仕様を改定する
- 外部依存やビルドツールを追加する

## Decisions

- **図の形式は Mermaid に統一する**  
  変更履歴が追いやすく、テキスト差分でレビューできるため。
  - *Alternative:* 画像ファイル → 差分管理が難しいため不採用。
- **既存章を増補し新章は最小限にする**  
  読者が迷わないよう、文脈は既存章内で強化する。
  - *Alternative:* 新規ドキュメントを複数追加 → 文脈が分断される。
- **Cockburn の概念は引用でなく要約で扱う**  
  ライセンスや著作権に配慮し、要点を噛み砕いて整理する。

## Risks / Trade-offs

- [文量が多くなり読み切れない] → 各章に「読む順」「要点サマリ」を追加
- [概念が抽象的で理解しづらい] → アプリの具体コード参照を必ず添える
