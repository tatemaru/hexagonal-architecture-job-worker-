## Why

バックエンドの実装は FastAPI/Python/Redis と DDD・ヘキサゴナルアーキテクチャを前提にしており、これらに不慣れなエンジニアにとって理解コストが高い。これを解消し、開発参加や保守がしやすい状態にするため、学習ガイド兼アーキテクチャ解説の技術書レベルのドキュメントを整備する。

## What Changes

- docs フォルダを新設し、バックエンド実装の読み方・構成・流れを丁寧に解説したドキュメントを追加する
- DDD/ヘキサゴナルアーキテクチャの概念説明と、本アプリにおける対応箇所を明示する
- マーメイド記法の ER 図を追加する

## Capabilities

### New Capabilities
- `backend-docs`: バックエンドの実装を理解するための技術書レベルのドキュメント（FastAPI/Python/Redis/DDD/ヘキサゴナルの基礎から本アプリへの対応付けまで）を提供する

### Modified Capabilities
- （なし）

## Impact

- 追加ファイル: `docs/` 以下のドキュメント
- 既存コードや API には影響しない
